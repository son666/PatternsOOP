Архитектура CRM "Учета заказов сервисного центра"

Архитектура будущей CRM была выбрана "Клиент/Сервер" т.к система в будущем должна отвечать следующим критериям:

1. Клиент должен работать в браузере, в связи с чем, CRM будет состоять из следующих частей:
	1.1 Web клиента (реализация с помощью React)
	1.2 Back реализация на Spring Boot
	Взаимодействие между клиентом и сервером будет происходить по средствам Rest запросов
2. Простота в развертывании и поддержке
3. Ограниченные мощностя
4. Безопасность
5. Система не планируется быть большой т.к по текущим наблюдениям основные функции это:
	1. Создание заказа с информацией о клинете и его оборудовании в ремонте
	2. В процессе ремонта добавляется список используемых запчастей и услуг, а так же мастер производивший ремонт
	3. Подсчет стоимости ремонта
	4. Ведение журнала мастеров
